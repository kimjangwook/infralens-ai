# Contributing

InfraLens is early-stage. Contributions should keep the first version small and useful for self-hosted CloudOps briefing.

Good first areas:

- AWS/GCP read-only scanner coverage
- Finding rules with clear evidence
- Slack/Notion export
- Japanese report templates
- Least-privilege policy examples

Avoid:

- Automatic cloud resource modification
- Raw log retention by default
- Broad platform rewrites before v0.1 is stable
